Guía para Contribuir: Pull Request (PR) a GitHub

Repositorio Destino: bvislao/white-list-antiphishing-peru

Sigue estos pasos para sugerir la adición de nuevos dominios.

1 Forkea el Repositorio

Crea una copia del repositorio original en tu propia cuenta de GitHub.

Ve a la página del repositorio (enlace) y haz clic en el botón "Fork" (Bifurcar) en la esquina superior derecha.

2 Clona tu Fork Localmente

Abre tu terminal (o Git Bash) y clona la copia (fork) que acabas de crear a tu computadora. Reemplaza [TU_USUARIO] con tu nombre de usuario de GitHub.

git clone https://github.com/[TU_USUARIO]/white-list-antiphishing-peru.git

3 Crea y Muévete a una Nueva Rama

Siempre trabaja en una nueva rama, no en main, para mantener tus cambios aislados.

cd white-list-antiphishing-peru
git checkout -b agregar-dominios-nuevos

(El nombre de la rama debe ser descriptivo, por ejemplo: agregar-dominio-banco-x)

4 Realiza tus Cambios

Edita el archivo relevante (probablemente un archivo `.json` o `.txt` con la lista de dominios). Asegúrate de que tu cambio sigue el formato y las reglas existentes del repositorio.

Ejemplo de Edición: Abre el archivo y añade el nuevo dominio.

5 Haz Commit y Sube los Cambios

Añade los archivos modificados, realiza un commit (guarda los cambios localmente) y luego súbelos a tu fork remoto en GitHub.

git add .
git commit -m "feat: Agregando nuevos dominios bancarios a la lista blanca"
git push origin agregar-dominios-nuevos

6 Abre el Pull Request (PR)

Ve a la página de tu fork en GitHub. Verás un botón verde que dice "Compare & pull request" (Comparar y solicitar pull).

  • Haz clic en este botón.
  • Revisa que la rama base sea main del repositorio original y la rama comparada sea tu nueva rama.
  • Escribe un título claro y una descripción detallada de tus cambios.
  • Haz clic en "Create pull request".

¡Listo! El mantenedor del repositorio revisará tu contribución.
